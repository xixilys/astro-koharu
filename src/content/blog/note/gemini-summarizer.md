---

## 🧠 为什么选 Gemini 3.0 Pro？

相比 GPT-4o 或 Claude 3.5 Sonnet，Gemini 3.0 Pro 有一个杀手锏特性：
**200 万 token 的超大上下文窗口 (Context Window)**。

这意味着我们可以直接把一整本 PDF 书籍扔给它，甚至几十篇论文一起扔进去，它都不会“忘记”前面的内容。

---

## 🛠️ 我的实现方案：`summarize_multiround.py`

为了充分利用这个特性，我和 **🌲树** 开发了一个 Python 脚本。
它的工作流程不是简单的“概括全文”，而是像人类专家一样进行 **多轮对话 (Multi-round Conversation)**：

### 第一轮：全景扫描 (Overview)
*   **Prompt**: "请阅读这篇论文，用 300 字总结它的核心贡献、创新点和实验结果。"
*   **目的**: 快速了解这篇论文在讲什么，是否值得细读。

### 第二轮：关键技术挖掘 (Technical Deep Dive)
*   **Prompt**: "针对文中提到的 CIM 宏单元设计，详细解释它的电路结构、存内计算原理以及与传统冯诺依曼架构的区别。"
*   **目的**: 针对我的研究方向（CIM 硬件加速），定向提取硬核技术细节。

### 第三轮：实验数据分析 (Data Extraction)
*   **Prompt**: "请提取论文中的实验数据表格，特别是关于能效比 (TOPS/W)、面积效率 (TOPS/mm²) 和准确率 (Accuracy) 的数据。"
*   **目的**: 把非结构化的文本转化为结构化的 JSON 数据，方便我做横向对比。

### 第四轮：批判性思考 (Critical Thinking)
*   **Prompt**: "这篇论文有哪些局限性？它的实验设置是否有漏洞？未来的改进方向是什么？"
*   **目的**: 培养批判性思维，不仅仅是接受信息，更是评估信息。

## 🚀 效果演示

现在，我只要把一堆 PDF 扔进文件夹，运行脚本：
```bash
python summarize_multiround.py --input ./papers --output ./summaries
```

几分钟后，我就能得到一份包含 **技术细节、数据对比、优缺点分析** 的高质量摘要报告。
而且因为是 **Gemini 3.0 Pro**，它甚至能看懂论文里的**电路图**和**架构图**！😲
